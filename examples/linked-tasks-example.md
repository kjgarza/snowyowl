# Example TASKS.md with Linked Specifications

This example demonstrates how to use linked task specifications.

- [ ] Implement remember me functionality

- [ ] Add indexes for performance
- [ ] Set up connection pooling

## Testing

- [ ] Add integration tests for authentication
- [ ] Improve unit test coverage to 80%
- [ ] Add E2E tests with Playwright

## Documentation

- [ ] Update API documentation
- [ ] Add setup guide for developers
- [ ] Create deployment documentation

## Notes

In this example:
- The first task links to `./specs/oauth2-implementation.md` which contains detailed requirements
- Other tasks are simple and don't need detailed specs
- You can mix both approaches in the same file
